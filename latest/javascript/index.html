MDK

\n The MDK API consists of two interfaces: MDK and Session. The\n MDK interface holds globally scoped APIs and state associated\n with the microservice. The Session interface holds locally\n scoped APIs and state. A Session must be used sequentially.\n\n There will typically be one MDK instance for the entire\n process, and one instance of the Session object per\n thread/channel/request depending on how the MDK is integrated\n and used within the application framework of choice.\n
